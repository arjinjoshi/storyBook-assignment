Git Commands that we learned today

Today we learned about the github and git. Not only that, We had also explored a lot of git commands by ourselves. Among them the command which fascinated me alot was ` git commit --amend -m "message" `. This command is generally used to rewrite the old comment(message). That means it will delete the old message that we have passed with the commit and put the new written one while using this command.
And if we haven't pushed the original commit yet, then we can use just `git push` command. But if we have already pushed the original commit to github then we must use `git push --force` command. Always make sure to use --force command on the branches instead of main.

And if we want to use revert the commit message we can do it via `git revert HEAD -m "message"`. It will gonna work in both the cases: even if we have pushed the code on github or we haven't pushed yet. Also I learned, we can replace the HEAD with hashkey of most recent commit that we can see if we use "git log -1 --oneline" and if we wanna see the full details we can use git log -5 --pretty=fuller then we can see the hashkey of each commit and if we want to revert multiple commits then make sure Revert the old one first

Every year, sacks of gold were carried up the stone steps to the Grand Hall, and every year the sacks came back lighter, as if the air itself had learned to steal. The ministers spoke in velvet sentences. Roads were announced before they were drawn. Schools were promised before they were planned. Bridges existed first as speeches, then as cracks.
